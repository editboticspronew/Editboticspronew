1. What is ClipWeave?

ClipWeave is a fully browser-based video editing platform designed around one core idea: let AI do the tedious work of reviewing, selecting, and cutting footage, so creators can focus on storytelling.

Users upload videos to organized projects, provide a natural-language prompt (e.g., "Extract the camera review section" or "Get all product demos"), and the platform identifies, clips, and merges the relevant segments Гўв‚¬вЂќ all without leaving the browser.

What makes the platform stand apart is its dual-system approach. Rather than forcing every video through a single pipeline, ClipWeave supports two complementary editing systems that each excel in different scenarios.

The platform's technology stack is chosen to maximize browser-side processing while using server-side AI only for intelligence tasks.

Step 1 Client-Side

Audio Extraction

FFmpeg WebAssembly extracts the audio track from the uploaded video file Гўв‚¬вЂќ no server upload required. The extraction runs entirely in the browser and produces a format compatible with the Whisper API.

Step 2 Server-Side

Speech-to-Text Transcription

The extracted audio is sent to the transcription API route, which forwards it to OpenAI Whisper. The response includes:

  • Full transcript text Гўв‚¬вЂќ the complete spoken content
  • Timestamped segments Гўв‚¬вЂќ each sentence/phrase with start and end times

Both are persisted to Firestore, so re-transcription is never needed. The UI shows a green chip when timestamps are available and a yellow chip when only plain text exists.

Step 3 Server-Side

AI Segment Analysis

The user enters a natural-language query describing what clips they want. The timestamped transcript segments are sent to GPT-4o-mini along with the query. The AI model returns a structured JSON response identifying which segments match, including:

  • Start and end timestamps for each relevant clip
  • Relevance score indicating how well each clip matches the query
  • Brief label for each identified clip
Step 4 Client-Side

Video Clipping & Merging

FFmpeg WASM cuts the original video at the identified timestamps. Multiple clips are automatically merged with configurable transition effects (15 transition types available, including crossfade, wipe, slide, and more). The final merged output can be previewed and downloaded Гўв‚¬вЂќ all without a server round-trip.

Best For

  • Any content where dialogue drives the narrative

6. Clip Generation & Auto-Merge

Both systems share the same final stage: browser-based video processing via FFmpeg WebAssembly. This is where the AI's decisions become actual video files.

Clipping Process

Once the AI identifies relevant segments with timestamps, each clip is cut from the original video file entirely client-side. No video data leaves the browser. The clipping engine handles:

  • Precise timestamp-based cutting
  • Multiple clips processed in sequence with progress tracking
  • Blob URL generation for instant preview

Auto-Merge with Transitions

After clipping, the system automatically merges all clips into a single output video. Users can choose from 15 transition effects between clips:

Category Transitions
BlendFade, Crossfade, Dissolve, Smoothup, Smoothdown

Audio transitions use crossfade to prevent abrupt pops. A fallback mechanism handles cases where video transitions fail Гўв‚¬вЂќ the system automatically retries with a simple concatenation to ensure output is always generated.

Editor Integration

Generated clips can be sent to the built-in timeline editor, where users can:

  • Reorder clips via drag-and-drop
  • Add text overlays with custom positioning and styling
  • Preview with the custom video player (subtitle overlays, custom controls)
  • Export the final composition Гўв‚¬вЂќ all rendered in-browser

8. Architecture Principles

1
Browser-First Processing All video rendering Гўв‚¬вЂќ clipping, merging, transitions, text overlays, and export Гўв‚¬вЂќ runs in the browser via FFmpeg WebAssembly. Servers handle only AI inference. This minimizes bandwidth, cost, and latency.
2
Analyze Once, Query Forever Both transcription and vision analysis are cached permanently in the database. A user can generate clips with different queries against the same video without re-processing Гўв‚¬вЂќ only the lightweight scoring and LLM call run again.
3
Layered Intelligence The system is modular: transcript is the base layer, vision is an optional enhancement, scoring is configurable, and AI selection adapts its prompt based on what data is available. Each layer can improve independently.
4
Token-Efficient AI Communication Raw vision data (thousands of frame-level detections) is aggregated into compact segment-level summaries before reaching the AI model. This keeps token usage predictable and prevents model overwhelm.
5
Graceful Degradation If vision analysis isn't configured or fails, the system seamlessly falls back to transcript-only mode. If video transitions fail during merge, it retries with simple concatenation. The user always gets output.
6
Progressive Enhancement Users start with the simplest workflow (upload ГўвЂ вЂ™ transcribe ГўвЂ вЂ™ clip) and can progressively enable advanced features (vision analysis, custom transitions, editor timeline) as their needs grow.
